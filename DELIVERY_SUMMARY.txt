API ENDPOINTS
================================================================================

BASE URL: http://localhost:8000

EMPLOYEES:
  POST   /api/employees              - Create employee
  GET    /api/employees              - Get all employees
  GET    /api/employees/{emp_id}     - Get specific employee
  DELETE /api/employees/{emp_id}     - Delete employee

ATTENDANCE:
  POST   /api/attendance             - Mark attendance
  GET    /api/attendance/{emp_id}    - Get employee attendance
  GET    /api/attendance             - Get all attendance records

SYSTEM:
  GET    /health                     - Health check
  GET    /docs                       - Interactive API documentation
  GET    /redoc                      - ReDoc documentation

================================================================================
QUICK START COMMANDS
================================================================================

1. Using Docker (Recommended):
   cd /Users/aadeshparija/Documents/Daily_code/hrms-lite
   docker-compose up --build

2. Manual Setup:
   Backend:
     cd backend && python3 -m venv venv && source venv/bin/activate
     pip install -r requirements.txt && python main.py

   Frontend (new terminal):
     cd frontend && npm install && npm start

3. Accessing the Application:
   Frontend:     http://localhost:4200
   Backend API:  http://localhost:8000
   API Docs:     http://localhost:8000/docs

VALIDATION RULES IMPLEMENTED
================================================================================

EMPLOYEE:
  - Employee ID: Required, unique, 1-50 characters
  - Full Name: Required, 1-100 characters
  - Email: Required, valid format, unique
  - Department: Required, 1-100 characters

ATTENDANCE:
  - Employee ID: Required, must exist in database
  - Date: Required, YYYY-MM-DD format
  - Status: Required, "Present" or "Absent"
  - Constraint: One record per employee per day
